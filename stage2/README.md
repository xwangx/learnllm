# 阶段② — MNIST 手写数字分类

四阶段学习路径第 2 步。从阶段①"4 个点的玩具"跨到真实数据集，
第一次面对：数据加载、mini-batch、训练/验证集、过拟合、评估指标、代码复用、GPU。

## 怎么跑

```bash
python stage2/01_mnist_mlp.py     # 全连接网络（阶段①的延续）
python stage2/02_mnist_cnn.py     # 卷积网络（同一训练循环，换模型）
```

第一次运行会自动下载 MNIST 到 `data/`（约 11MB，不进 git）。默认用 GPU。

## 实测结果

| 模型 | 测试准确率 | 训练5轮后过拟合间隙 |
|------|-----------|-------------------|
| MLP（全连接） | **97.53%** | +0.0127 |
| CNN（卷积） | **99.11%** | +0.0068 |

CNN 不仅更准，过拟合也更轻；它错分的还都是真的长得像的数字（3↔8、4↔9、9↔7）。

```
common.py     数据加载 + 训练/评估函数 —— 两个模型共享
```

## 这一阶段的新概念

### mini-batch（小批量）
阶段①是 4 个点一次性全喂。真实数据 6 万张，太大放不下也没必要。
`DataLoader` 每次只吐一小批（这里 64 张），算一次梯度更新一次。一轮(epoch)走完所有批。

### 训练集 / 验证集 / 测试集
- **训练集**：拿来学习、更新参数。
- **验证集**：从训练数据切出来、训练时不参与学习，只用来观察"在没见过的数据上行不行"。
- **测试集**：全程不碰，最后才用一次，是最诚实的成绩。

### 过拟合 (overfitting)
模型在训练集上越来越好，但在验证集上不再提升甚至变差——它在"死记硬背"训练样本，
而不是学到泛化规律。**判断方法**：训练准确率明显高于验证准确率（代码里打印的"过拟合间隙"）。
**缓解手段**：本阶段用了 **Dropout**（训练时随机丢弃一部分神经元，逼模型别依赖个别神经元）。
其他常见手段：早停、加更多数据、数据增强、正则化。

### 为什么图像要用 CNN
- MLP 把图片拉平成 784 个独立像素，丢掉了空间结构（哪些像素挨在一起）。
- CNN 用小窗口(kernel)在图上滑动，捕捉局部图案（边缘、笔画），且参数在全图共享。
- 所以 CNN 参数更少、还能识别"图案出现在哪都算数"，对图像天然更强。

### 交叉熵损失 & Adam
- **CrossEntropyLoss**：多分类标准损失（内部含 softmax）。它的输出层梯度同样是"预测−真值"，
  和阶段①推导 BCE 时见到的结论一脉相承。
- **Adam**：比纯 SGD 更聪明的优化器，会自适应每个参数的步长，收敛更快。本质仍是"沿梯度下降"。
