# 阶段① — 用 XOR 理解神经网络训练

四阶段学习路径的第 1 步。目标：亲眼看懂"训练"在底层做了什么。

## 怎么跑

```bash
python stage1/01_xor_numpy.py     # 纯 numpy，手写一切
python stage1/02_xor_pytorch.py   # 同一个网络，用 PyTorch autograd
```

两个脚本同样的网络（输入2 → tanh隐藏层4 → sigmoid输出1）、同样的 XOR 数据、
同样的超参（lr=1.0, 5000 轮），最后都把 4 个 XOR 点完美分类。

## 核心收获

**训练的本质是一个循环，每轮做 4 件事：**

1. **前向** —— 用当前参数算出预测
2. **算 loss** —— 预测离真值差多少（这里用二元交叉熵 BCE）
3. **反向** —— 求出"每个参数该往哪个方向调"（梯度）
4. **更新** —— 参数朝梯度反方向走一小步（SGD）

**numpy 版 vs PyTorch 版，差别只在第 3、4 步：**

| 步骤 | numpy 版（手写） | PyTorch 版 |
|------|-----------------|-----------|
| 前向 | 手写矩阵乘 + 激活 | `model(X)` |
| 算 loss | 手写 BCE 公式 | `nn.BCELoss()` |
| **反向** | **手推链式法则求每个梯度** | **`loss.backward()` 一行** |
| **更新** | **手写 4 行 `param -= lr*grad`** | **`optimizer.step()` 一行** |

PyTorch 的 autograd 自动化的，正是 numpy 版里最烧脑的"手推反向传播"。
但概念一模一样——框架没有变魔术，只是替你算了导数。

## 为什么选 XOR

XOR 不是线性可分的（画不出一条直线分开 0 和 1），所以**单层线性模型学不会**，
必须要带非线性激活的隐藏层。这把"为什么需要多层 + 反向传播怎么穿过隐藏层"讲透了。

## 数值梯度检查（仅 numpy 版）

`01_xor_numpy.py` 里用有限差分 `(L(p+ε) - L(p-ε)) / 2ε` 逼近梯度，
对照手推的解析梯度——相对误差约 1e-9，证明手推公式没写错。
这是调试反向传播的黄金方法。
