# 阶段④ 设计：用 LoRA 微调 Qwen2.5，注入数据治理风格

> 四阶段学习路径第 4 步（收官）。承接阶段③（从零训 GPT），这次站在预训练模型肩上做微调。

## 重要前提（已与用户确认）

用户有一个数据治理个人 wiki（`Z:\work\wiki`，2415 篇 md，约 6.8MB 中文）。
目标"让模型更懂数据治理"，**最佳工程方案其实是 RAG**（检索增强生成）——
微调不擅长注入事实知识，擅长风格/口吻/格式。用户已知晓，仍选择**学习微调流程**。

因此本阶段定位：**学一遍完整、通用的微调真功夫**（数据准备 → LoRA → 训练 → 对比），
产物是"说话带数据治理味儿"的模型。事实准确性不是目标。

## 核心思路

拿预训练好的 Qwen2.5-0.5B，用 wiki 正文继续做"预测下一个字"的训练
（与阶段③同一目标），但只用 **LoRA** 微调极小一部分参数。

## LoRA（本阶段新概念）

全量微调要更新全部 ~5 亿参数，慢且占显存。LoRA 冻结原模型，
只在旁边插入一组小的低秩"适配器"参数（通常 <1% 总量），只训练它们。
效果接近全量微调，但又快又省，且适配器可插拔、仅几 MB。是当今微调的事实标准。

## 产出物

```
stage4/
├── prepare_data.py   读取 Z:\work\wiki 的 md，去 frontmatter、拼接、按 token 切块
├── finetune.py       加载 Qwen2.5-0.5B + 套 LoRA + Trainer 训练 + 保存适配器
├── compare.py        同一提示对比 "原始模型 vs 微调后" 的生成
└── README.md         讲清微调、LoRA、与阶段③的对比
```

## 数据准备

- 遍历所有 `.md`，去掉 frontmatter（--- 块），留正文。
- 拼成大语料，用 Qwen tokenizer 切成固定长度块（block_size=512）。
- 约 6.8MB → 数千训练块，足够看到风格迁移。

## 训练

- transformers + peft 套 LoRA（target 注意力投影层 q/k/v/o），Trainer 训练。
- 基座：Qwen/Qwen2.5-0.5B（从 HuggingFace 下载，约 1GB；HF 直连可用）。
- 默认 GPU（RTX 5090）。保存 LoRA 适配器（几 MB），非整模型。
- 超参起点：lr 2e-4，epoch 2~3，batch 视显存，LoRA r=8/alpha=16/dropout 0.05。

## 成功标准

- 训练 loss 下降。
- compare.py 肉眼可辨：给开头（如"数据资产建设"），微调后续写明显更贴 wiki 术语风格
  （本体、血缘、数据标准、owner、Policy 等），原始模型更泛泛。
- LoRA 适配器能保存、能加载叠加到基座生成。
- README 讲清 LoRA 及"微调 vs 从零训"。

## 范围之外（YAGNI）

- 不追求事实准确（RAG 的职责，已说明）。
- 不做量化/多卡/混合精度等工程优化（单卡 0.5B 够轻）。
- 不做指令式问答微调（本阶段为续写式风格微调）。

## 依赖

transformers 4.54、accelerate 1.10（已装）；peft、datasets（本阶段安装）。
基座模型从 HuggingFace 下载。
