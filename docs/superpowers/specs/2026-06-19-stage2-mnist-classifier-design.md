# 阶段② 设计：MNIST 手写数字分类

> 四阶段学习路径第 2 步。承接阶段①（XOR 从零训练），第一次面对真实数据集。
> 后续：③ 从零训小 LLM、④ 微调现成模型。

## 目标

从"4 个点的玩具"跨到"真实数据集"，第一次接触真实训练绕不开的东西：
数据加载、mini-batch、训练/验证集划分、过拟合、评估指标、代码复用、用 GPU。

## 产出物

```
stage2/
├── common.py            数据加载 + 通用训练/评估函数（两模型共享）
├── 01_mnist_mlp.py      全连接网络（阶段①的直接延续）
├── 02_mnist_cnn.py      卷积网络（同一训练循环，只换模型结构）
└── README.md            对比 MLP vs CNN，讲清新概念
```

`common.py` 体现本阶段新主题：把可复用代码抽出来。模型脚本只定义网络，
训练/评估逻辑共享——体会"训练循环是通用的"。

## 数据

- `torchvision` 自动下载 MNIST（6万训练 + 1万测试，28×28 灰度，10 类）到 `data/`（已 gitignore）。
- `DataLoader` 做 mini-batch（batch_size=64）——相对阶段①"4点全量"的关键升级。
- 从训练集切出验证集（5万训练 / 1万验证），用于监控过拟合。

## 两个模型

- **MLP**：28×28 拉平成 784 → Linear(128) + ReLU → Linear(10)。预期验证准确率 ~97%。
- **CNN**：两层 Conv + 池化 → Linear(10)。预期 ~99%。注释讲清卷积为何适合图像。
- 损失：交叉熵 (CrossEntropyLoss)，多分类标准（softmax + CE，输出层误差同样是"预测−真值"，呼应阶段①彩蛋）。
- 优化器：Adam（比纯 SGD 收敛快；注释说明它与 SGD 的关系）。
- 默认用 GPU（RTX 5090）。代码演示 `.to(device)` 把模型与数据搬上显卡。

## 训练流程 & 评估（common.py）

每个 epoch：训练一遍 → 验证集评估 → 打印 训练loss / 验证loss / 训练准确率 / 验证准确率。
- 评估指标：准确率 (accuracy)。训练后打印若干被错分样本。
- **过拟合演示（已确认加入）**：同时打印训练准确率与验证准确率。训练准确率明显高于验证准确率即过拟合。
  在 MLP 上故意展示这一现象，并演示一个缓解手段（dropout 或早停），讲清这是真实训练的日常问题。

## 成功标准

- MLP 验证准确率 ≥ 97%，CNN ≥ 99%。
- 两脚本可直接 `python stage2/0x_xxx.py` 运行：自动下载数据、自动用 GPU。
- README 讲清：mini-batch 是什么、为什么需要验证集、过拟合长什么样、CNN 为何强于 MLP。

## 范围之外（YAGNI）

- 不做数据增强、不做学习率调度、不做超参搜索。
