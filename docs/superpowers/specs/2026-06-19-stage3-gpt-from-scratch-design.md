# 阶段③ 设计：从零手写一个 GPT，生成莎士比亚

> 四阶段学习路径第 3 步。承接阶段②（MNIST 分类），第一次接触 Transformer / LLM。
> 后续：④ 微调现成模型。

## 核心思路

喂给模型一大段莎士比亚文本，训练它做一件事：**看前面的字符，预测下一个字符**。
学好后给一个开头，让它一个字一个字往下接（自回归生成）——这就是 GPT 的全部原理。

## 产出物

```
stage3/
├── gpt.py        从零手写的 GPT 模型（自注意力、多头、Transformer block）
├── train.py      数据加载 + 训练循环 + 保存权重 (checkpoint)
├── generate.py   加载训练好的权重，生成文本
├── input.txt     莎士比亚语料（约 1MB，自动下载）
└── README.md     讲清 tokenize、注意力、自回归生成
```

本阶段第一次使用保存/加载模型（`torch.save`/`torch.load`）——训练几分钟，
训完存盘，之后随时 `generate.py` 生成、不用重训。补上阶段②故意留的一课。

## Tokenize（字符级）

- 列出文本里所有不同字符（约 65 个），每个映射成一个整数。"hi" → [20, 21]。
- 训练时随机截取固定长度（block_size=256）的文本块；输入是前 n 个字符，目标是右移一位的下一个字符。

## 模型结构（全部手写，本阶段重点）

从下到上：
- **Token Embedding + 位置 Embedding**：字符整数 → 向量；并编码"每个字在第几位"。
- **N 个 Transformer Block**，每块含：
  - **多头自注意力 (Multi-Head Self-Attention)**：Query/Key/Value、缩放点积注意力分数、
    **因果 mask**（不能偷看未来字符）。注释讲透。
  - **前馈网络 (FeedForward)** + 残差连接 + LayerNorm（Pre-LN 结构）。
- **输出层 (lm_head)**：映射到词表大小，得到下一个字符的 logits。
- 损失：交叉熵（预测的下一个字 vs 真实下一个字）。

### 规模（已确认）
- n_layer=6, n_head=6, n_embd=384, block_size=256。约 1000 万参数。
- 优化器 AdamW，lr≈3e-4，含 dropout(0.2) 防过拟合。
- 默认 GPU（RTX 5090），几分钟训完。

## 生成（自回归）

给定起始字符串 → 不断"预测下一个字符的概率分布 → 采样一个 → 接到末尾 → 再预测"，
循环生成指定长度。用温度 (temperature) 控制随机性。

## 成功标准

- 训练 loss 明显下降（字符级莎士比亚一般可到 ~1.5）。
- 生成文本"像莎士比亚"：有人名、对白格式（如 `FIRST CITIZEN:`）、英文单词大体成形
  （非完美英语，但能看出学到了风格与结构）。
- 权重能存盘、能重新加载生成。
- README 把注意力机制讲清楚。

## 范围之外（YAGNI）

- 不用 BPE/子词分词（字符级即可）。
- 不做分布式/混合精度等工程优化（单卡够快）。
- 不追求生成完美英语（规模和数据决定了上限，重点是理解机制）。

## 环境

Windows，Python 3.13.5，torch 2.7.1+cu128（CUDA 可用），RTX 5090。
