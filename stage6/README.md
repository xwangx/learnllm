# 阶段⑥ — 从零手写 ViT，并用吴恩达方法论分析

学习路径第 6 阶段。多模态里"视觉"那一半的核心：用 Transformer 处理图像。
是阶段②(图像分类) + 阶段③(Transformer) 的合体，并用吴恩达《ML Yearning》的框架做诊断。

> 🖼️ **图解版**：[vit_explainer.html](vit_explainer.html)（浏览器打开）——含数据流图、ViT vs CNN 对比图、偏差/方差诊断、"为什么数据饥渴"图解。

## 核心洞见

**ViT = 把阶段③那个 Transformer 拿来处理图像。** 区别只有两点：
1. 输入不是文字 token，而是把图片切成小方块(patch)，每块当一个 token；
2. 注意力**不加因果 mask**（看图前后都能看，不像 GPT 生成文本要"不能看未来"）。

```
图片(3×32×32) → 切16个/64个patch → 每块线性嵌入 → 加[CLS]+位置嵌入
  → N层Transformer编码器 → 取[CLS] → 分类头 → 10类
```

## 怎么跑

```bash
python stage6/get_cifar.py          # 多源轮询下载 CIFAR-10（国内可达镜像，断点续传）
python stage6/train_vit.py --model cnn   # 训练 CNN 基线
python stage6/train_vit.py --model vit   # 训练 ViT
```

> 数据下载：torchvision 默认从 toronto.edu 下 CIFAR-10，国内常连不上。
> `get_cifar.py` 改用 fast.ai 的 S3 镜像（图片文件夹格式），断点续传、重试到成功。

## 实测结果（CIFAR-10，各训 30 轮）

| 模型 | 参数量 | 测试准确率 |
|------|--------|-----------|
| CNN 基线 | 2.20M | **90.8%** |
| ViT（从零） | 2.69M | **79.7%** |

**关键观察：ViT 参数更多，准确率反而低 11 个百分点。** 这不是 bug，是 ViT 的本质特点。

放进整个学习路径看（架构演进）：
| 模型 | 数据集 | 测试准确率 |
|------|--------|-----------|
| MLP（阶段②） | MNIST | 97.5% |
| CNN（阶段②） | MNIST | 99.1% |
| CNN（阶段⑥） | CIFAR-10 | 90.8% |
| ViT（阶段⑥） | CIFAR-10 | 79.7% |

## 吴恩达式诊断（偏差 vs 方差）

对标 CIFAR-10 人类水平 ≈ 94%（误差 6%）：

| | 训练误差 | 验证误差 | 可避免偏差(训练−人类) | 方差(验证−训练) |
|---|---|---|---|---|
| **CNN** | 2.3% | 9.6% | **−3.7%**（几乎为0） | 7.2% |
| **ViT** | 12.7% | 21.7% | **+6.7%** | 9.0% |

- **CNN**：训练误差极低（甚至低于人类水平估计），可避免偏差≈0，问题主要是**方差/过拟合**——加数据/增强/正则即可继续提升。
- **ViT**：**既有可避免偏差(6.7%)又有方差(9.0%)**——它连训练集都没很好地拟合（欠拟合），同时又过拟合。这正是"缺乏图像先验"的表现。

## 为什么 ViT 打不过 CNN？（本阶段的核心一课）

**CNN 自带"图像先验"（归纳偏置）**：卷积天生假设"局部相关""平移不变"——这些正是图像的规律。
所以 CNN 用有限数据就能学得又快又好。

**ViT 几乎没有这种先验**：它把图片当成一串 patch，用纯注意力"从零"学习哪些 patch 该关注。
表达能力更强、上限更高，但**代价是数据饥渴**——需要海量数据（或大规模预训练）才能学到这些规律。

CIFAR-10 只有 5 万张图，对 ViT 远远不够，所以它输给了 CNN。这与 ViT 原论文的结论一致：
**ViT 只有在超大规模预训练（如 3 亿张图）后，才会反超 CNN。** 这也正是阶段④"迁移学习/预训练
为何重要"的延续——多模态大模型里的视觉编码器，无一例外都是大规模预训练出来的。

## 误差分析（吴恩达式：看模型错在哪）

最易混淆的类别对：
- CNN：猫→狗 90 次、狗→猫 80 次（语义相近的动物，情有可原）
- ViT：猫→狗 170 次、狗→猫 123 次（同样的混淆，但错得更多——能力不足的体现）

两者都最容易混"猫和狗"，符合直觉；ViT 的错误量整体更大。

## 和多模态的关系

ViT 就是多模态大模型（如 Qwen-VL、LLaVA）里**视觉编码器**的原型：
把图片变成一串 token，再用投影层对齐到 LLM 的 token 空间，LLM 就能"看图说话"。
理解了 ViT，你就理解了多模态的"视觉"那一半。
