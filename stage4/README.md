# 阶段④ — 用 LoRA 微调 Qwen2.5，注入数据治理风格

四阶段学习路径收官。前三阶段从零造轮子理解原理，这一阶段学最实用的玩法：
**站在预训练模型肩上做微调**。

## ⚠️ 重要前提（诚实说明）

目标是"让模型更懂数据治理"。**最佳工程方案其实是 RAG（检索增强生成）**——
微调不擅长注入事实知识，擅长风格/口吻/格式。本阶段是为了**学习微调的完整流程**，
产物是"说话带数据治理味儿"的模型；**它不保证事实准确**（那是 RAG 的职责）。
学到的流程（数据准备 → LoRA → 训练 → 对比）是完全通用的真功夫。

## 怎么跑

```bash
python stage4/prepare_data.py   # 把 wiki 的 md 整理成训练语料 corpus.txt
python stage4/finetune.py       # LoRA 微调（约 3 分钟），保存适配器
python stage4/compare.py        # 对比 原始模型 vs 微调后
python stage4/compare.py --prompt "数据质量管理" --tokens 160
```

模型放在 `stage4/Qwen2.5-0.5B/`（约 1GB，需自行下载，见仓库历史说明）。

## 实测结果

- 基座 Qwen2.5-0.5B（4.95 亿参数），**LoRA 只训练 0.22%（108 万参数）**，其余全冻结。
- 训练 loss 2.63 → 1.79，约 3 分钟训完（RTX 5090）。
- LoRA 适配器仅 **20 MB**（对比完整模型 988 MB）。

对比示例（提示"数据资产建设"）：
- **原始模型**：跑偏到选择题、金币、珠宝，抓不住数据治理语境。
- **微调后**：立刻进入 wiki 的语态——markdown 文档结构、`**来源**`/`**核心主题**` 字段、
  业务文档术语，甚至复刻了 wiki 的格式特征。

诚实评价：风格迁移明显（尤其文档结构和领域词汇如实体/属性/操作），
但 0.5B 小模型 + 轻量微调能力有限，生成仍会飘。要更好效果可换更大模型或更多训练。

## 核心概念

### 微调 (Fine-tuning) vs 从零训练（阶段③）
- 阶段③：从零造空白 GPT，从头学语言 → 只会模仿莎士比亚。
- 阶段④：拿已懂中文/世界的预训练模型，用少量数据"调味"。
  **这就是为什么现在没人从零训大模型**——站在巨人肩上又快又好。

### LoRA（低秩适配）
全量微调要更新全部参数，慢且占显存。LoRA 冻结原模型，
只在目标层（这里是注意力的 Q/K/V/O 投影）旁插入一对小矩阵，只训练它们：
- 只训练 <1% 参数 → 又快又省显存
- 适配器只有几十 MB，可插拔、可叠加到基座
- 效果接近全量微调

是当今微调的事实标准。补充：**QLoRA = 4-bit 量化 + LoRA**，能在消费级显卡微调几十 B 大模型。

### 微调 vs RAG（什么时候用哪个）
| 需求 | 用什么 |
|---|---|
| 改变模型的风格、语气、格式、行为 | 微调 |
| 让模型基于我的文档准确答疑、能给出处、随文档更新 | RAG |
| 两者兼顾 | 微调调风格 + RAG 保事实 |
